Help & Documentation

Getting Started

Welcome to the Dot1xer Supreme Configurator! This tool helps you generate 802.1X configurations for various network vendors.

  • Configuration Tab: Follow a step-by-step process to configure and generate device-specific 802.1X settings.

Vendor-Specific Notes

  • Cisco IOS-XE: Ensure 'aaa new-model' is enabled before applying 802.1X settings.
  • Cisco NX-OS: Requires 'feature dot1x' to be enabled.
  • ArubaOS: Use AAA profiles for 802.1X authentication.
  • Juniper: Configure 802.1X under the 'protocols dot1x' hierarchy.
  • Extreme: Use 'configure netlogin' for 802.1X settings.
  • Arista: Similar to Cisco IOS, but uses 'dot1x' commands directly.

Troubleshooting

  • Config Not Applying: Verify RADIUS server reachability and shared key.
  • Authentication Fails: Check client certificates and VLAN assignments.
